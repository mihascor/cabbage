# Проект: Cabbage

## Общее описание

Cabbage — это локальное десктоп-приложение на Python с GUI (PyQt6), предназначенное для сбора, хранения и анализа данных Московской биржи (MOEX).

Приложение выступает как аналитическая панель (dashboard) с возможностью запуска парсеров, записи данных в SQLite и последующей визуализации.

---

## Архитектура проекта

### 1. GUI (PyQt6)

* Точка входа: `app/main.py`
* Функции:

  * запуск скриптов (парсеров)
  * отображение логов выполнения
  * управление процессами через QThread
* GUI напрямую вызывает Python-модули (без subprocess)

---

### 2. Скрипты (бизнес-логика)

* Расположение: `script/`
* Основной модуль: `parser_moex.py`
* Функции:

  * асинхронный парсинг данных с MOEX (Playwright)
  * обработка и нормализация данных
  * передача результатов в БД

---

### 3. База данных (SQLite)

* Путь: `C:\cabbage\data\cabbagedb.db`
* Используется для хранения:

  * списка инструментов (futures_list)
  * результатов парсинга (open_interest_record)

---

### 4. Логирование

* Путь: `C:\cabbage\logs\`
* Формат: файл + вывод в GUI
* Используется для отладки и мониторинга выполнения

---

### 5. Сборка

* Инструмент: PyInstaller
* Режим: `--noconsole`
* Результат: standalone `.exe` в `dist/`
* Особенности:

  * требуется явное подключение модулей (hidden-import)
  * Playwright браузеры не входят в сборку автоматически

---

## Текущая реализация парсера

### Технологии:

* asyncio
* Playwright (headless Chromium)
* SQLite

### Логика:

1. Чтение списка инструментов из БД
2. Параллельный запуск задач (Semaphore)
3. Парсинг страницы MOEX
4. Извлечение open interest
5. Валидация данных
6. Запись в БД (с предварительной очисткой)

---

## Особенности реализации

* Асинхронная архитектура (async/await)
* Ограничение параллелизма (`MAX_CONCURRENT`)
* Обработка ошибок с возможностью полной остановки
* Используется глобальный путь к браузерам:
  `PLAYWRIGHT_BROWSERS_PATH`

---

## Ограничения

* Жёсткие пути (C:\cabbage...)
* Зависимость от установленного Playwright (браузеры)
* Требуется пересборка `.exe` при изменении кода
* Windows-only окружение

## Как со мной работать (для ChatGPT)

Важно учитывать:

* проект асинхронный
* используется Playwright
* сборка через PyInstaller может ломать зависимости
* GUI не использует subprocess
* код должен быть совместим с Windows

При ответах:

* давать точечные изменения (не переписывать всё без необходимости)
* избегать решений, ломающих сборку
* Если надо поправть код частично, показывай в формате БЫЛО / СТАЛО + строка-якорь для поиска
* При Написании кода, важные блоки, прокоментировать на русском языке.
