# PROJECT: Cabbage

## 1. Краткое описание

Cabbage — локальное десктоп-приложение (Python + PyQt6) для загрузки, хранения и анализа данных MOEX.

Функции:

* сбор рыночных данных (парсинг MOEX)
* хранение в SQLite
* простая аналитика и визуализация в GUI

---

## 2. Архитектура (слои)

### GUI (PyQt6)

* entrypoint: `app/main.py`
* запускает бизнес-логику
* отображает логи
* управление через QThread (без subprocess)

### Core / Scripts

* папка: `script/`
* основной модуль: `parser_moex.py`
* отвечает за получение и обработку данных

### Data Layer

* SQLite: `C:\cabbage\data\cabbagedb.db`
* хранит:

  * инструменты
  * исторические данные (open interest и др.)

### Logging

* `C:\cabbage\logs\`
* логирование + вывод в GUI

---

## 3. Парсер MOEX

Стек:

* asyncio
* Playwright (Chromium)
* SQLite

Логика:

1. загрузка списка инструментов из БД
2. параллельный запуск задач (Semaphore)
3. парсинг MOEX страниц
4. извлечение данных
5. запись в БД (очистка/апдейт)

Особенности:

* async/await архитектура
* ограничение параллелизма
* остановка при ошибках
* PLAYWRIGHT_BROWSERS_PATH используется явно

---

## 4. Сборка

* PyInstaller (`--noconsole`)
* output: `dist/`
* особенности:

  * hidden-imports обязательны
  * Playwright браузеры не включены в сборку

## 7. Файлы проекта (реестр)

### GUI

* `app/main.py` — entrypoint GUI

### UI
Путь C:\cabbage\app\ui
* `parser_tab.py` - Вкладка "Парсер"
* `table_tab.py` - Вкладка "Таблица"
* `instrument_window.py` - # Окно аналитики инструмента

### Widgets 
Путь C:\cabbage\app\ui\widgets
* `oi_graph_widget.py` - Рисует график ОИ для выбранного инструмента.
* `price_graph_widget.py` - # Рисует график цены для выбранного инструмента.

### Scripts
Путь C:\cabbage\script\
* `parser_moex.py` — Основной парсер
* `append_history.py` — Добавление данных в историю open_interest_history из open_interest_record с UPSERT (вставка или обновление)
* `load_table.py` — Загрузка данных из open_interest_record для отображения в таблице
* `instrument_repository.py` - # SQL и подготовка данных для графиков ОИ и цены.

### DB

* `data/cabbagedb.db` — SQLite база данных

### Конфигурация и служебные файлы

Путь: `C:\cabbage\data\`

* `signals.json` — состояние кнопок колонки «Выбор» в GUI
* `last_run.json` — дата и время последнего запуска парсера
* `last_price_update.json` — время последнего обновления цен акций через API
